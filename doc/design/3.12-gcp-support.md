# GCP (Google Cloud Platform) 云厂商支持实现

## 概述

本文档记录在 redc 和 redc-gui 中添加 GCP 云厂商支持的过程，供后续维护参考。

GCP 与其他云厂商的关键区别在于：**使用服务账号 JSON 凭据文件路径，而非 AK/SK 密钥对**。

---

## 1. 后端配置 (mod/config.go)

### 1.1 添加 GCP 配置结构体

文件: `mod/config.go`

```go
Google struct {
    Credentials string `yaml:"GOOGLE_APPLICATION_CREDENTIALS" env:"GOOGLE_APPLICATION_CREDENTIALS"`
    Project     string `yaml:"project"`
    Region      string `yaml:"region"`
} `yaml:"google"`
```

### 1.2 获取凭据函数

GCP 不使用传统的 AK/SK 模式，因此 `GetProviderCredentials` 函数**不需要添加 GCP case**。

GCP 通过环境变量 `GOOGLE_APPLICATION_CREDENTIALS` 传递 JSON 凭据文件路径，Terraform Provider 会自动读取该环境变量。

---

## 2. Provider 配置 (mod/deployment_executor.go)

### 2.1 生成 Provider 配置

GCP 模板**不需要**通过 redc 生成 provider 配置，因为：

1. GCP 使用服务账号文件，而非环境变量传递凭据
2. 模板的 `versions.tf` 中已经包含了 provider 定义

如果需要在 `GenerateProviderConfig` 中添加 GCP 支持（用于未来自定义场景）:

```go
case "gcp":
    return &ProviderConfig{
        Name:    "google",
        Source:  "hashicorp/google",
        Version: "~> 7.0",
        Config: map[string]string{
            "project": project,
            "region":  region,
        },
    }, nil
```

## 3. GUI 凭据页面 (app.go)

### 3.1 获取凭据配置

文件: `app.go`

在 `GetProvidersConfig` 函数中添加 GCP 凭据字段:

```go
{
    Name: "Google Cloud",
    Fields: map[string]string{
        "credentials": conf.Providers.Google.Credentials,  // 注意：不使用 maskValue，直接显示
        "project":     conf.Providers.Google.Project,
        "region":      conf.Providers.Google.Region,
    },
    HasSecrets: map[string]bool{},  // credentials 是文件路径，不是 secret
},
```

**关键点**：
- `credentials` 字段不使用 `maskValue()` 包装，直接显示原始路径
- `HasSecrets` 为空 map，不显示🔒图标

### 3.2 保存凭据配置

文件: `app.go`

在 `SaveProvidersConfig` 函数中添加:

```go
case "Google Cloud":
    if v, ok := fields["credentials"]; ok && v != "" {
        conf.Providers.Google.Credentials = v
    }
    if v, ok := fields["project"]; ok {
        conf.Providers.Google.Project = v
    }
    if v, ok := fields["region"]; ok {
        conf.Providers.Google.Region = v
    }
```

```javascript
    project: '项目 ID',
    region: 'Region',

```

### 4.2 字段显示顺序

```javascript
const fieldOrder = {
    // ... 其他云厂商
    'Google Cloud': ['credentials', 'project', 'region'],
};
```

### 4.3 密码字段判断

在 `isSecretField` 函数中移除 `credentials`:

```javascript
function isSecretField(key) {
    const secrets = ['accessKey', 'secretKey', 'secretId', 'clientId', 'clientSecret', 
                     'subscriptionId', 'tenantId', 'user', 'tenancy', 'fingerprint', 'apiKey'];
    return secrets.includes(key);
}
```

这样 `credentials` 字段会以明文显示，而非密码类型。

```hcl

  }

}
```

## 6. 配置文件示例

### 6.1 config.yaml

```yaml
providers:
  google:
    GOOGLE_APPLICATION_CREDENTIALS: "/path/to/your-service-account.json"
    project: "your-project-id"
    region: "us-central1"
```

### 6.2 环境变量

```bash
# GCP 使用标准环境变量
GOOGLE_APPLICATION_CREDENTIALS=/path/to/your-service-account.json
```

## 7. 注意事项

1. **凭据类型**: GCP 使用服务账号 JSON 文件，**不是** AK/SK 密钥对
2. **环境变量**: 使用标准的 `GOOGLE_APPLICATION_CREDENTIALS`（不是 `GOOGLE_CREDENTIALS`）
3. **凭据显示**: 由于是文件路径而非密钥，前端应显示为明文，不添加🔒图标
4. **Provider 版本**: 建议使用固定版本如 `7.21.0`
5. **Terraform Provider**: GCP 使用 `hashicorp/google` 而非 `google-beta`
6. **项目 ID**: GCP 必须填写项目 ID (project)

## 9. 与其他云厂商的关键区别

| 特性 | AWS/阿里云/腾讯云等 | GCP |
|------|---------------------|-----|
| 凭据类型 | AK/SK 密钥对 | 服务账号 JSON 文件 |
| 环境变量 | 各自独立 | `GOOGLE_APPLICATION_CREDENTIALS` |
| 密码字段 | 需要 maskValue | 直接显示路径 |
| HasSecrets | 标记为 secret | 空 map |
| 获取凭据函数 | 需要添加 case | 不需要（使用文件路径） |
