# 3.35 CLI JSON 输出支持

## 背景

redc CLI 原有输出为面向人类阅读的日志格式（gologger），不适合脚本化调用和自动化集成。需要让所有命令支持 `--output json` 参数，输出结构化 JSON 数据。

## 设计原则

1. **向后兼容** — 默认输出行为不变，仅在 `--output json` 时切换为 JSON
2. **统一格式** — 成功输出 `{"data": ...}`，错误输出 `{"error": "..."}`
3. **静默模式** — JSON 模式下抑制所有 gologger 输出，仅向 stdout 输出一行 JSON
4. **不影响 GUI** — GUI 不使用 CLI 命令，JSON 输出仅作用于 CLI 入口

## 实现方案

### 核心文件：`cmd/output.go`

定义全局变量 `outputFormat` 和辅助函数：

```go
var outputFormat string

type JSONOutput struct {
    Data  interface{} `json:"data,omitempty"`
    Error string      `json:"error,omitempty"`
}

func IsJSON() bool              // 判断是否 JSON 模式
func PrintJSON(data interface{}) // 输出 {"data": ...}
func PrintJSONError(err error)   // 输出 {"error": "..."}
func PrintJSONMessage(msg string) // 输出 {"data": {"message": "..."}}
func MustJSON(data interface{}, err error) // 有错输出 error，无错输出 data
```

### 注册方式：`cmd/root.go`

在 root 命令注册持久化 flag：

```go
rootCmd.PersistentFlags().StringVarP(&outputFormat, "output", "o", "", "Output format (json)")
```

### 各命令适配模式

每个命令在关键输出点检查 `IsJSON()`：

```go
if IsJSON() {
    PrintJSON(data)
    return
}
gologger.Info().Msgf("原有日志输出")
```

## 覆盖范围

| 文件 | 命令 | JSON 输出内容 |
|------|------|---------------|
| root.go | version | `{"version": "x.y.z"}` |
| actions.go | ps | Case 列表数组 |
| actions.go | stop/start/kill/rm | `{"action": "...", "id": "..."}` |
| actions.go | status | terraform state（outputs + resources） |
| actions.go | change | `{"action": "change", "id": "..."}` |
| plan.go | plan | `{"action": "plan", "name": "...", "id": "..."}` |
| plan.go | run | `{"action": "run", "name": "...", "state": "..."}` |
| compose.go | up/down | `{"message": "compose up/down completed"}` |
| compose.go | config | 原输出（inspect 直接打印） |
| pull.go | pull | `{"template": "...", "status": "pulled"}` |
| pull.go | search | 模板列表数组（直接序列化 `mod.Search` 结果） |
| tmpl.go | image ls | `ListLocalTemplates()` 结果数组 |
| tmpl.go | image rm | `{"message": "template removed: ..."}` |
| exec.go | exec | 命令执行（交互模式不支持 JSON） |
| exec.go | cp | `{"action": "upload/download", "src": "...", "dest": "..."}` |
| init.go | init | 每个模板初始化结果数组 `[{"dir": "...", "status": "ok/failed"}]` |

### 不覆盖的命令

- `logs` — 流式日志输出，JSON 模式无意义
- `mcp stdio/sse` — 服务器进程，本身就是 JSON-RPC
- `completion` — Shell 补全脚本生成

## 使用示例

```bash
# 列出所有场景
redc ps --output json

# 启动并获取结构化结果
redc run ecs --output json

# 搜索模板
redc search aliyun --output json

# 脚本中使用
CASE_ID=$(redc run ecs -o json | jq -r '.data.id')
redc status $CASE_ID -o json | jq '.data.outputs'
```
