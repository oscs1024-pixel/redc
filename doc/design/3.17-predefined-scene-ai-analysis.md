# 预定义场景 AI 错误分析功能设计文档

## 概述

本文档记录了预定义场景（Cases）页面的 AI 错误分析功能开发，用于在场景创建失败时自动分析错误原因并提供解决方案。

---

## 1. 功能需求

### 1.1 背景

用户在预定义场景页面创建场景时，如果创建失败，需要手动查看错误日志才能了解失败原因。使用 AI 自动分析错误可以大幅提升用户体验。

### 1.2 功能目标

1. 错误信息持久化显示 - 错误发生后不会因页面刷新而消失
2. AI 错误分析 - 点击按钮自动分析错误原因并提供解决方案
3. 与自定义部署页面 UI 保持一致

---

## 2. 技术实现

### 2.1 错误日志级别修改

**文件**: `mod/tf.go`, `mod/case.go`

将关键错误日志从 `gologger.Debug()` 改为 `gologger.Error()`，确保非 Debug 模式下也能输出：

```go
// mod/tf.go
err = te.Plan(ctx, o...)
if err != nil {
    gologger.Error().Msgf("场景创建失败: %v", err)
    return err
}
```

### 2.2 前端错误状态管理

**文件**: `frontend/src/components/Cases/Cases.svelte`

#### 2.2.1 持久化错误存储

使用 `window` 对象存储错误，实现页面刷新后仍能保留：

```javascript
// 使用 window 对象持久化错误
window.__persistentError = window.__persistentError || null;
let persistentError = $state(null);

// 同步 persistentError
$effect(() => {
    persistentError = window.__persistentError;
});

function setPersistentError(err) {
    window.__persistentError = err;
    persistentError = err;
}

function getPersistentError() {
    return persistentError;
}
```

#### 2.2.2 刷新保护

在 `refresh()` 函数中添加保护，防止刷新清除错误状态：

```javascript
export async function refresh() {
    // Don't refresh if there's an error status
    if (createStatus === 'error') {
        return;
    }
    // Also don't refresh if currently creating/initializing
    if (createStatus === 'creating' || createStatus === 'initializing') {
        return;
    }
    // ... refresh logic
}
```

#### 2.2.3 AI 分析 Key 跟踪

使用独立变量跟踪当前 AI 分析的 key，确保关闭错误后仍能看到分析结果：

```javascript
let currentAIKey = $state(null);

// 在开始 AI 分析时设置
currentAIKey = caseId;
```

### 2.3 后端 AI 分析函数

**文件**: `app.go`

```go
func (a *App) AnalyzeCaseError(caseId, errorMessage, provider, templateName string) error {
    profile, err := redc.GetActiveProfile()
    if err != nil || profile.AIConfig == nil {
        return fmt.Errorf("请先配置 AI 服务")
    }

    client := ai.NewClient(
        profile.AIConfig.Provider,
        profile.AIConfig.APIKey,
        profile.AIConfig.BaseURL,
        profile.AIConfig.Model,
    )

    systemPrompt := `你是一个云资源部署专家助手...`
    // ... 构建 prompt

    err = client.ChatStream(ctx, messages, func(chunk string) error {
        runtime.EventsEmit(a.ctx, "ai-case-error-chunk", map[string]string{
            "caseId": caseId,
            "chunk":  chunk,
        })
        return nil
    })

    runtime.EventsEmit(a.ctx, "ai-case-error-complete", map[string]interface{}{
        "caseId":  caseId,
        "success": err == nil,
    })

    return err
}
```

### 2.4 前端事件监听

**文件**: `Cases.svelte`

```javascript
onMount(async () => {
    await refresh();
    
    // 监听 AI chunk
    EventsOn('ai-case-error-chunk', (data) => {
        const { caseId, chunk } = data;
        if (caseId && chunk) {
            aiAnalysisResult[caseId] = (aiAnalysisResult[caseId] || '') + chunk;
            aiAnalysisResult = { ...aiAnalysisResult };
        }
    });
    
    // 监听 AI 完成
    EventsOn('ai-case-error-complete', (data) => {
        const { caseId, success } = data;
        if (caseId) {
            aiAnalyzing[caseId] = false;
            aiAnalyzing = { ...aiAnalyzing };
            aiAnalysisCompleted[caseId] = true;
            aiAnalysisCompleted = { ...aiAnalysisCompleted };
        }
    });
});
```

## 4. 事件机制

| 事件名 | 方向 | 数据格式 | 说明 |
|--------|------|----------|------|
| `ai-case-error-chunk` | 后端 → 前端 | `{caseId, chunk}` | AI 返回的文本片段 |
| `ai-case-error-complete` | 后端 → 前端 | `{caseId, success}` | AI 分析完成 |

## 6. 注意事项

1. **Svelte 5 响应式**: 修改状态后需要使用 `state = { ...state }` 触发响应式更新

2. **事件命名**: 使用 `ai-case-error-*` 前缀，与自定义部署的 `ai-deployment-error-*` 区分

3. **持久化策略**: 使用 `window.__persistentError` 确保错误信息在组件重绘后不丢失

4. **刷新保护**: 在 `refresh()` 中检查错误状态，避免刷新清除错误显示

5. **AI 配置依赖**: 功能依赖用户在设置中配置 AI 服务
