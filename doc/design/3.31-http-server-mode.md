# HTTP Server 模式设计文档

## 功能概述

redc-gui 支持以 HTTP Server 模式运行，使前端 Svelte GUI 可通过浏览器访问，无需安装桌面应用。

## 两种使用方式

### 1. 命令行无窗口模式
```bash
redc-gui server --port=8899 --host=127.0.0.1 --token=mytoken
```
- 无桌面窗口，仅启动 HTTP 服务
- 启动时打印访问地址和 Token

### 2. GUI 内嵌模式
在设置页面 > HTTP Server 区域开启，可同时使用桌面应用和浏览器访问。

## 技术架构

### 后端

**事件分发统一化**：将原先散落在各处的 `runtime.EventsEmit(a.ctx, ...)` 调用统一收口到 `a.emitEvent(name, data)` 方法。该方法根据运行模式分别向：
- Wails 桌面模式：调用 `runtime.EventsEmit`
- HTTP 模式：通过 SSE Hub 广播给所有连接的浏览器客户端

**HTTP Server** (`http_server.go`):
- `POST /api/call` — 反射分发器，通过方法名调用 App 的任意公开方法
- `GET /api/events` — SSE 端点，客户端长连接接收实时事件
- `GET /api/auth` — Token 验证端点
- `GET /*` — 静态文件服务（SPA fallback 到 index.html）

**认证**：Bearer Token（`Authorization: Bearer <token>` 或 `?token=<token>`）

### 前端

**HTTP Transport Polyfill** (`src/lib/http-transport.js`):

在浏览器模式下（检测到 `window.runtime` 不存在），安装两个 shim：

1. `window['go']['main']['App']` — 使用 `Proxy` 对象，任意属性访问返回一个函数，调用时发 `POST /api/call` 请求
2. `window.runtime` — 实现 EventsOn/EventsOff/Environment/Window 等 Wails API，其中事件订阅通过 SSE 实现，Window 控制为 no-op

**模式检测**：`main.js` 在挂载 Svelte 应用前检测 `window.runtime` 是否存在：
- 存在 → Wails 模式（不安装 polyfill）
- 不存在 → HTTP 模式（安装 polyfill，设置 `window.__redcWebMode__ = true`）

**窗口控件**：`App.svelte` 中 `isWebMode` 变量控制是否显示 Windows 专属的最小化/最大化/关闭按钮。

## 关键文件

| 文件 | 作用 |
|------|------|
| `http_server.go` | HTTP 服务器、SSE Hub、API 反射分发器 |
| `app.go` | `emitEvent()` 统一事件分发方法 |
| `app_system.go` | StartHTTPServer/StopHTTPServer/GetHTTPServerStatus/GetHTTPServerConfig/SetHTTPServerConfig |
| `main.go` | `server` 子命令检测，`runServerMode()` |
| `mod/config.go` | GUISettings 中的 HTTP Server 配置字段 |
| `frontend/src/lib/http-transport.js` | 前端 HTTP/SSE polyfill |
| `frontend/src/main.js` | polyfill 安装入口 |
| `frontend/src/components/Settings/Settings.svelte` | HTTP Server 配置 UI |

## SSH Terminal 说明

SSH Terminal 的输出通过 `a.emitEvent("terminal-output-xxx", data)` 推送。由于所有事件都经过 SSE Hub 转发，Terminal 在 HTTP 模式下可以正常工作（输出流通过 SSE 到达浏览器，输入通过 `WriteToTerminal` API 调用发回后端）。

## 限制与注意事项

1. 将 HTTP Server 绑定到公网 IP 时，请确保 Token 足够复杂
2. 多个浏览器客户端同时连接时，所有事件（包括其他用户触发的操作）都会广播给所有客户端
3. `redc-gui server` 模式下，文件对话框（SelectFile、SelectDirectory）在无桌面环境中无法使用
