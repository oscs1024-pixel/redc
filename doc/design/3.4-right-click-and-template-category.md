## 1. 右键菜单禁用功能

### 1.1 概述

在设置页面添加右键菜单禁用开关，用户可选择启用或禁用右键菜单功能。

### 1.2 后端实现

**文件**: `app.go`

```go
// App 结构体添加字段
type App struct {
    // ... 其他字段 ...
    disableRightClick bool
}

// 设置右键禁用状态
func (a *App) SetDisableRightClick(enabled bool) error {
	a.mu.Lock()
	defer a.mu.Unlock()
	a.disableRightClick = enabled
	return nil
}

// 获取右键禁用状态，默认返回 true（禁用）
func (a *App) GetDisableRightClick() bool {
	a.mu.Lock()
	defer a.mu.Unlock()
	// 默认返回 true，Go 的 bool 默认值是 false，需要特殊处理
	if !a.disableRightClick {
		return true
	}
	return a.disableRightClick
}
```

### 1.3 前端实现

**文件**: `frontend/src/App.svelte`

```svelte
<script>
  import { GetDisableRightClick, SetDisableRightClick } from '../wailsjs/go/main/App.js';
  
  let rightClickDisabled = $state(true);  // 默认禁用
  
  // 初始化时加载设置
  rightClickDisabled = await GetDisableRightClick();
  
  // 注册右键处理
  const handleContextMenu = (e) => {
    if (rightClickDisabled) {
      e.preventDefault();
      return false;
    }
  };
  document.addEventListener('contextmenu', handleContextMenu);
</script>
```

### 1.4 设置页面

**文件**: `frontend/src/components/Settings/Settings.svelte`

```svelte
<script>
  import { SetDisableRightClick } from '../../../wailsjs/go/main/App.js';
  
  let { rightClickDisabled = $bindable(false) } = $props();
  let rightClickSaving = $state(false);
  
  async function handleToggleRightClick() {
    const nextValue = !rightClickDisabled;
    rightClickSaving = true;
    try {
      await SetDisableRightClick(nextValue);
      rightClickDisabled = nextValue;
    } catch (e) {
      console.error('Failed to toggle right click:', e);
    } finally {
      rightClickSaving = false;
    }
  }
</script>

<!-- 右键菜单设置 -->
<div class="bg-white rounded-xl border border-gray-100 p-4 sm:p-5">
  <div class="flex flex-col sm:flex-row sm:items-center justify-between gap-3 sm:gap-0">
    <div>
      <div class="text-[13px] sm:text-[14px] font-medium text-gray-900">{t.disableRightClick}</div>
      <div class="text-[11px] sm:text-[12px] text-gray-500 mt-1">{t.disableRightClickDesc}</div>
    </div>
    <button
      class="relative inline-flex h-6 w-11 items-center rounded-full transition-colors disabled:opacity-50 disabled:cursor-not-allowed"
      class:bg-emerald-500={rightClickDisabled}
      class:bg-gray-300={!rightClickDisabled}
      onclick={handleToggleRightClick}
      disabled={rightClickSaving}
    >
      <span class="inline-block h-4 w-4 transform rounded-full bg-white transition-transform"
        class:translate-x-6={rightClickDisabled}
        class:translate-x-1={!rightClickDisabled}
      ></span>
    </button>
  </div>
</div>
```

### 1.6 关键要点

1. **Go bool 默认值**: Go 的 bool 默认值是 `false`，所以需要在 `GetDisableRightClick()` 中特殊处理，默认返回 `true`

2. **加载时序**: 右键处理函数需要在应用初始化时注册，且在 `rightClickDisabled` 状态加载完成后执行

3. **状态绑定**: 使用 `$bindable` 让设置页面的状态与父组件双向同步

## 2. 本地模板分类功能

### 2.1 概述

本地模板页面添加标签页分类，支持：预定义模板、自定义模板、Userdata模板、Compose模板。

### 2.2 分类规则

| 分类 | 条件 |
|------|------|
| 自定义模板 | 模板名称包含 `/base-templates/` |
| Userdata模板 | 模板名称包含 `/userdata-templates/` |
| Compose模板 | 模板名称包含 `/compose-templates/` |
| 预定义模板 | 除上述外的其他模板 |

### 2.3 前端实现

**文件**: `frontend/src/components/LocalTemplates/LocalTemplates.svelte`

```svelte
<script>
  // 标签页状态
  let templateTab = $state('all');
  
  // 分类函数
  function getTemplateType(tmpl) {
    const name = tmpl.name || '';
    
    if (name.includes('/base-templates/')) {
      return 'custom';
    }
    if (name.includes('/userdata-templates/')) {
      return 'userdata';
    }
    if (name.includes('/compose-templates/')) {
      return 'compose';
    }
    return 'preset';
  }
  
  // 根据标签页过滤
  let filteredLocalTemplates = $derived(localTemplates
    .filter(t => {
      if (templateTab !== 'all') {
        const type = getTemplateType(t);
        if (templateTab !== type) return false;
      }
      if (localTemplatesSearch) {
        const search = localTemplatesSearch.toLowerCase();
        return t.name.toLowerCase().includes(search) ||
          (t.description && t.description.toLowerCase().includes(search));
      }
      return true;
    })
    .sort((a, b) => a.name.localeCompare(b.name)));
</script>

<!-- 标签页 UI -->
<div class="flex gap-2 border-b border-gray-200 mb-4">
  <button
    class="px-4 py-2 text-[13px] font-medium transition-colors {templateTab === 'all' ? 'text-blue-600 border-b-2 border-blue-600' : 'text-gray-600 hover:text-gray-900'}"
    onclick={() => templateTab = 'all'}
  >
    {t.allTemplates || '全部'}
  </button>
  <button
    class="px-4 py-2 text-[13px] font-medium transition-colors {templateTab === 'preset' ? 'text-blue-600 border-b-2 border-blue-600' : 'text-gray-600 hover:text-gray-900'}"
    onclick={() => templateTab = 'preset'}
  >
    {t.presetTemplates || '预定义模板'}
  </button>
  <!-- 其他标签页... -->
</div>
```

### 2.5 分类工具函数

**文件**: `frontend/src/lib/userdataTemplates.js`

```javascript
// 按分类分组模板
export function getGroupedTemplates(templates) {
  if (!Array.isArray(templates)) return {};
  const groups = {};
  for (const template of templates) {
    const cat = template.category || 'other';
    if (!groups[cat]) groups[cat] = [];
    groups[cat].push(template);
  }
  return groups;
}

export const userdataCategoryNames = {
  basic: '基础环境',
  ai: 'AI 应用',
  security: '安全工具',
  vulhub: '漏洞环境',
  other: '其他'
};
```

### 2.6 关键要点

1. **分类依据**: 使用模板名称中的路径信息判断分类，因为后端返回的模板对象中 `path` 字段为空

2. **标签页设计**: 参考专项模块页面设计，使用下划线指示当前选中项

3. **搜索过滤**: 搜索功能和标签页过滤需要同时生效

4. **国际化**: 需要在 i18n.js 中添加对应的翻译key
