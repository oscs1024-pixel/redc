# AI 流式响应与命令执行超时优化

## 问题 1: AI 流式响应超时丢失内容

### 现象
AI Agent 回复到一半时，连接超时报错：
```
AI 分析失败: failed to send request: Post ".../chat/completions": net/http: timeout awaiting response headers
```
页面上已显示的部分内容消失。

### 根因
1. `ResponseHeaderTimeout` 设为 30s，复杂 Agent 提示词（含工具调用历史）需要模型更多思考时间
2. 前端 `AIChat.svelte` 在收到错误时直接清除 `streamingContent`，未保留已接收内容

### 修复
1. **client.go**: `ResponseHeaderTimeout` 从 30s 提升至 120s（proxy 和非 proxy 两个 Transport）
2. **AIChat.svelte**: 两处错误路径添加部分内容保存逻辑：
   - `EventsOn('ai-chat-complete')` 的 `success:false` 分支
   - `sendMessage()` 的 `catch` 块
   - 保存为正常消息，内容末尾附 `⚠️ 响应中断` 标记

### 涉及文件
- `mod/ai/client.go`: 行 43、56
- `frontend/src/components/AI/AIChat.svelte`: ai-chat-complete handler、sendMessage catch
- `frontend/src/lib/i18n.js`: 新增 `aiChatStreamInterrupted` key

## 问题 2: exec_command 长任务超时无法续期

### 现象
某些命令（大型编译、go install、apt 安装）在 Linux 上执行时间超过默认超时（300s），被强制终止。

### 方案
命令超时时不直接终止，而是弹出选择框让用户确认：
- **继续等待**: 延长 10 分钟，可多次续期
- **终止命令**: 立即 kill 进程

### 实现

#### 后端 (mcp.go)
```go
type ExecTimeoutAskFunc func(conversationId, toolName, command string) bool

// MCPServer 新增字段
execTimeoutAsk sync.Map // conversationId -> ExecTimeoutAskFunc

// toolExecCommand 超时处理
for {
    select {
    case <-time.After(timeout):
        if askFunc != nil && askFunc(convId, "exec_command", cmd) {
            continue // 用户选择继续，重新进入 select 等待
        }
        // 用户选择终止或无回调
        session.Signal(ssh.SIGTERM)
    case result := <-done:
        return result
    }
}
```

#### 前端回调 (app_ai_chat.go)
复用已有的 `ask_user` 前端交互机制（`ai-agent-ask-user` 事件 + `askUserChannels`）。

```go
mcpServer.RegisterExecTimeoutAsk(conversationId, func(convId, toolName, command string) bool {
    // 发送 ai-agent-ask-user 事件到前端
    // 等待用户在 askUserChannels 中的回复
    // 返回 true（继续）或 false（终止）
})
```

#### 参数传递
通过 `_conversation_id` 隐藏参数注入到 ExecuteTool 的 args 中：
```go
args["_conversation_id"] = conversationId
result := mcpServer.ExecuteTool(toolName, args)
delete(args, "_conversation_id")
```

### 涉及文件
- `mod/mcp/mcp.go`: ExecTimeoutAskFunc 类型、Register/Unregister、toolExecCommand/toolExecUserdata 超时循环
- `app_ai_chat.go`: runAgentLoop 注册回调、ExecuteTool 注入 _conversation_id
