# AI Chat 多轮对话功能设计文档

## 概述

在 RedC GUI 中新增 **AI Chat** 标签页，提供类 ChatGPT 风格的多轮对话界面。支持 4 种对话模式（生成模板、推荐场景、成本优化、自由对话），后端流式输出，对话历史本地持久化。

原有的「AI 集成」标签页（单轮查询 + MCP 管理）**保持不变**，AI Chat 是独立的新功能。

## 2. 后端设计

### 2.1 数据类型

```go
// app_ai_chat.go
type AIChatMessage struct {
    Role    string `json:"role"`    // "user" 或 "assistant"
    Content string `json:"content"`
}
```

### 2.2 AIChatStream 函数签名

```go
func (a *App) AIChatStream(conversationId, mode string, messages []AIChatMessage) error
```

参数说明：
- `conversationId`：前端生成的唯一 ID（用于隔离并发请求的事件）
- `mode`：`"free"` / `"generate"` / `"recommend"` / `"cost"`
- `messages`：完整的历史对话（role + content），包含本次新消息

### 2.3 各 Mode 系统 Prompt 逻辑

| mode | system prompt | 特殊处理 |
|------|--------------|----------|
| `free` | `FreeChatSystemPrompt % langPrompt` | 无 |
| `generate` | `TemplateGenerationSystemPrompt + "\n\n" + langPrompt` | 无 |
| `recommend` | `TemplateRecommendationSystemPrompt % (templateList, langPrompt)` | 注入本地模板列表 |
| `cost` | `CostOptimizationSystemPrompt % langPrompt` | 调用 `gatherRunningCasesInfo()` 将运行中场景信息拼入最后一条 user 消息 |

**语言检测**：调用 `a.GetLanguage()`，`"en"` 时 `langPrompt = "Please reply in English"`，否则 `"请用中文回复"`。

### 2.4 流式事件

通过 Wails 事件系统向前端推送：

```go
// 每个流式 chunk
runtime.EventsEmit(a.ctx, "ai-chat-chunk", map[string]string{
    "conversationId": conversationId,
    "chunk":          chunk,
})

// 完成后（成功或失败）
runtime.EventsEmit(a.ctx, "ai-chat-complete", map[string]interface{}{
    "conversationId": conversationId,
    "success":        true/false,
})
```

超时：5 分钟（`context.WithTimeout`）。

### 2.5 gatherRunningCasesInfo 辅助函数

从 `app_ai_chat.go` 中提取，逻辑与 `app_ai.go` 中的 `AICostOptimization` 相同：

1. 遍历当前 project 的所有 cases，筛选 `StateRunning`
2. 对每个 case 调用 `redc.TfStatus(c.Path)` 获取 Terraform 状态
3. 调用 `costCalculator.CalculateCost()` 估算月度费用
4. 返回格式化的 Markdown 字符串 + 运行中 case 数量

## 3. 前端设计

### 3.1 组件文件

**路径**：`frontend/src/components/AI/AIChat.svelte`

**Props**：
- `t` — i18n 翻译对象（来自 App.svelte `$derived`）
- `onTabChange` — 切换标签页回调（备用）

### 3.2 状态管理

```js
let mode = $state('free');          // 当前对话模式
let messages = $state([]);          // 完整消息列表
let inputText = $state('');         // 输入框文本
let isStreaming = $state(false);    // 是否正在流式输出
let currentConversationId = $state(''); // 当前对话 UUID
let streamingContent = $state(''); // 正在流式输出的内容（不入 messages）
let error = $state('');
let successMessage = $state('');
let messagesContainer = $state(null); // 消息容器 DOM 引用（用于自动滚动）
```

### 3.3 消息结构

```js
{
  id: string,        // generateId() 生成
  role: 'user' | 'assistant',
  content: string,
  timestamp: number,
  mode: string       // 消息产生时的 mode，用于"保存模板"按钮显示逻辑
}
```

### 3.4 4 种模式

| id | 标签 key | 说明 |
|----|---------|------|
| `free` | `aiChatFreeChat` | 自由对话，通用助手 |
| `generate` | `aiChatGenTemplate` | 生成 Terraform 模板，assistant 消息显示「保存模板」按钮 |
| `recommend` | `aiChatRecommend` | 推荐本地模板 |
| `cost` | `aiChatCostOpt` | 成本优化分析 |

切换 mode 时：清空消息列表，插入对应 mode 的欢迎语（assistant 消息），重置流式状态。

### 3.5 发送消息流程

1. 将用户消息追加到 `messages`
2. 生成 `conversationId = generateId()`，设置 `isStreaming = true`
3. 将当前 `messages`（role + content）传给后端 `AIChatStream(convId, mode, chatMessages)`
4. 监听 `ai-chat-chunk` 事件 → 检查 `conversationId` 匹配 → `streamingContent += chunk`
5. 监听 `ai-chat-complete` 事件 → `success` 时将 `streamingContent` 作为 assistant 消息入列 → 清空 `streamingContent`，`isStreaming = false`

`conversationId` 校验防止多请求并发时 chunk 污染当前对话。

### 3.6 流式显示

- streaming 期间，在消息列表末尾渲染一个虚拟 assistant 消息框
- 内容显示 `streamingContent`，末尾加脉冲光标：`<span class="inline-block w-1.5 h-4 bg-rose-500 animate-pulse">`
- 无内容时（等待首个 chunk）显示 spinner + `t.aiChatStreaming`

### 3.7 自动滚动

```js
$effect(() => {
  if (streamingContent || messages.length) {
    scrollToBottom();
  }
});

function scrollToBottom() {
  if (messagesContainer) {
    requestAnimationFrame(() => {
      messagesContainer.scrollTop = messagesContainer.scrollHeight;
    });
  }
}
```

### 3.8 本地持久化

- 键名：`redc-ai-chat-state`
- 值：`JSON.stringify({ mode, messages })`
- 写入时机：每次消息追加完成后（`ai-chat-complete`）、mode 切换后、新建对话后
- 读取时机：`onMount`，若无历史则插入欢迎消息

> 原因：`App.svelte` 使用 `{#key activeTab}` 会在切换 tab 时销毁组件，localStorage 保证对话历史跨 tab 切换和应用重启后不丢失。

### 3.9 Action 按钮

每条 assistant 消息底部显示：

| 按钮 | 条件 | 行为 |
|------|------|------|
| 复制内容 | 始终显示 | `navigator.clipboard.writeText(content)` |
| 保存模板 | 仅 `mode === 'generate'` | `parseTemplateMarkdown(content)` 解析后调用 `SaveTemplateFiles` |

### 3.10 parseTemplateMarkdown

复用自 `AIIntegration.svelte`，将 AI 输出的 Markdown 按 `### filename` 分割，提取 `.json/.tf/.tfvars/.md/.sh/.yaml/.yml` 文件内容，去除代码块标记。

### 3.11 键盘快捷键

`Ctrl/Cmd + Enter` 发送（与 AIIntegration.svelte 保持一致）。

## 8. 注意事项

1. **并发防护**：每次发送生成新 `conversationId`，事件处理器检查 `data.conversationId === currentConversationId`，防止旧请求的 chunk 污染新对话。
2. **`{#key activeTab}`**：App.svelte 中此块包裹所有页面，切换 tab 时组件会被销毁并重建，必须依赖 localStorage 持久化状态。
3. **cost 模式首次消息**：后端会将运行中场景的成本信息自动拼接到用户消息前，无需前端额外处理。
4. **保存模板按钮**：仅在 `mode === 'generate'` 时出现，且基于 `msg.mode`（消息产生时的 mode），切换 mode 后旧消息不会错误显示保存按钮。
5. **无依赖原则**：样式全部使用 TailwindCSS 工具类，未引入 DaisyUI 等组件库，避免主题冲突。
