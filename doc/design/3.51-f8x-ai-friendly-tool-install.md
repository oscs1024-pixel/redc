# f8x AI 友好化工具安装改造

## 问题背景

f8x 是渗透测试工具集脚本（10,000+ 行 bash），包含 119 个 `Pentest_*_Install` 安装函数，但只有约 20 个工具暴露了独立 CLI flag（如 `-nmap`、`-python3`）。

**核心问题**：nuclei、masscan、httpx、ffuf、sqlmap 等最常用渗透工具没有独立安装选项，只能通过套件批量安装：
- `-ka` (TypeA)：38 个信息收集/扫描工具
- `-kb` (TypeB)：27 个漏洞利用工具
- `-kc` (TypeC)：14 个 AD/横向移动工具
- `-kd` (TypeD)：18 个辅助工具

AI Agent 遇到"安装 nuclei"时，尝试 `./f8x -nuclei` 失败（无此 flag），不知道 nuclei 在 `-kb` 套件里，即使知道也要安装 26 个不相关工具。

## 解决方案

### 1. 工具映射表 `F8X_TOOL_LIST`

在 f8x 脚本中新增管道分隔的工具映射表，每行格式：`工具名|安装函数|依赖等级`

```
F8X_TOOL_LIST="
nuclei|Pentest_Nuclei_Install|0
nmap|Pentest_nmap_Install|0
masscan|Pentest_masscan_Install|0
httpx|Pentest_httpx_Install|0
...（共 117 个工具）
"
```

依赖等级定义：
- 0: 仅基础环境（wget 下载二进制）
- 1: 需要 Python（pip install）
- 2: 需要 Go（go install）
- 3: 需要 Python + Go

### 2. `-install <工具名>` 精确安装

```bash
F8x_Install_Tool(){
    local tool_name=$(echo "$1" | tr 'A-Z' 'a-z')
    local entry=$(echo "$F8X_TOOL_LIST" | grep "^${tool_name}|" | head -1)
    # 解析 func 和 dep_level
    # 根据 dep_level 自动安装依赖（Py_Check/GO_Check）
    # 调用对应安装函数
}
```

在 case 分支中，`-install` 设置标志位 `_f8x_install_next=1`，下一个参数作为工具名传入。

### 3. `--list-tools` JSON 目录

输出所有工具的 JSON 数组，供 AI Agent 解析：

```json
[
  {"name":"nuclei","function":"Pentest_Nuclei_Install","deps":"basic"},
  {"name":"nmap","function":"Pentest_nmap_Install","deps":"basic"},
  ...
]
```

也支持 `--list-tools table` 输出人类可读的表格格式。

### 4. `--search <关键词>` 模糊搜索

按工具名模糊匹配，输出匹配的工具及其依赖信息：

```
$ ./f8x --search scan
  appinfoscanner  (deps: basic      func: Pentest_AppInfoScanner_Install)
  fscan           (deps: basic      func: Pentest_fscan_Install)
  masscan         (deps: basic      func: Pentest_masscan_Install)
  Found 3 tool(s). Use ./f8x -install <tool> to install.
```

### 5. 无 root 权限兼容

`--list-tools` 和 `--search` 是只读操作，在 `Main()` 函数（含 root 检查）之前拦截并提前退出，不需要 root 权限即可使用。

## Bash 兼容性

- 不使用 `declare -A`（关联数组），因为 macOS 默认 bash 3.2 不支持
- 使用管道分隔字符串 + `grep` 查找，兼容 bash 3.x+
- f8x 实际只在 Linux 上运行（有 `mac_Check` 守卫），但映射表和查询功能兼容所有平台

## 参数传递机制

f8x 主循环是 `for cmd in $@`，不支持 `shift`。解决方案：
- `-install` 设置 `_f8x_install_next=1` 标志
- 循环下次迭代检测到标志后，将当前参数作为工具名传入 `F8x_Install_Tool`
- `--search` 和 `--list-tools` 同理使用 `_f8x_search_next` 和 `_f8x_list_next`

## RedC 集成更新

### DeployAgentSystemPrompt

更新 Step 5.1 中的 f8x 引导：

```
1. exec_userdata(template_name='f8x-bash') 安装 f8x
2. ./f8x --search <关键词> 搜索工具
3. ./f8x -install <工具名> 精确安装
4. ./f8x --list-tools 查看完整目录
重要：使用 -install 而非 -ka/-kb 套件安装
```

### AgentSystemPrompt

工作原则 7 更新为引导使用 `-install` 和 `--search`。

### f8x-bash case.json

模板描述更新为明确标注 117 个工具和三种新命令。

## 效果

改造前：Agent 安装 nuclei → 尝试 `./f8x -nuclei` 失败 → 多轮试错 → 最终失败
改造后：Agent 安装 nuclei → `./f8x --search nuclei` → `./f8x -install nuclei` → 成功
