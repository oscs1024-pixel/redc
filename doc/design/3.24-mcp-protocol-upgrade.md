# 3.24 MCP 协议升级：2024-11-05 → 2025-11-25

## 背景

RedC GUI 内置了一个手写的 MCP（Model Context Protocol）服务器，用于让 Claude Desktop、Cursor 等 AI 工具通过标准协议操作 RedC 的模板、场景等资源。

原协议版本为 `2024-11-05`，已是旧版。2025-11-25 版本是当前最新 Current 版，引入了 Streamable HTTP transport、版本协商、通知机制等改进。

---

## 实现文件

- **`mod/mcp/mcp.go`** — MCP 服务器全部逻辑（协议、工具、资源、HTTP transport）

---

## 核心改动

### 1. 版本常量与版本协商

```go
const MCPVersion = "2025-11-25"

var supportedVersions = []string{"2025-11-25", "2024-11-05"}
```

`initialize` 请求时解析客户端的 `protocolVersion`：
- 如果客户端请求的版本在 `supportedVersions` 中，则回同版本（向下兼容）
- 否则回服务端最新版本

这样旧版客户端（如配置了 `2024-11-05` 的工具）仍可正常使用。

### 2. 通知处理

2025-11-25 规范要求客户端在 `initialize` 响应后发送 `notifications/initialized` 通知，服务端**不应**对通知返回响应。

```go
case "notifications/initialized", "notifications/cancelled":
    return nil  // 通知不回响应
```

在 STDIO 和 HTTP transport 中均检查 `resp == nil`，为 nil 时跳过响应发送（STDIO）或返回 `202 Accepted`（HTTP）。

### 3. InitializeResult 增加 instructions

```go
type InitializeResult struct {
    ProtocolVersion string             `json:"protocolVersion"`
    Capabilities    ServerCapabilities `json:"capabilities"`
    ServerInfo      ServerInfo         `json:"serverInfo"`
    Instructions    string             `json:"instructions,omitempty"`  // 新增
}
```

`instructions` 是可选字段，客户端可将其展示给用户或 AI，描述服务器用途。

### 4. Streamable HTTP 端点（新）

新增 `/mcp` 端点，同时支持：
- **POST** — 接收 JSON-RPC 请求/通知，返回 `application/json` 响应，通知返回 `202 Accepted`
- **GET** — 打开 SSE 流，供服务端主动推送消息（目前只发 connected 注释保持连接）

```
POST http://localhost:8089/mcp   <- 主端点（新）
GET  http://localhost:8089/mcp   <- SSE 流（新）
```

### 5. 保留旧端点（向后兼容）

`/message`（POST）和 `/sse`（GET/POST）端点保留不变，确保已配置旧端点的客户端不受影响。

### 6. CORS 统一处理

所有端点通过 `setCORSHeaders` 函数统一设置：

```go
w.Header().Set("Access-Control-Allow-Origin", "*")
w.Header().Set("Access-Control-Allow-Methods", "GET, POST, OPTIONS")
w.Header().Set("Access-Control-Allow-Headers", "Content-Type, Accept, MCP-Session-Id")
w.Header().Set("Access-Control-Expose-Headers", "MCP-Session-Id")
```

OPTIONS preflight 请求返回 `204 No Content`。

## 端点总览

| 端点 | 方法 | 说明 |
|------|------|------|
| `/mcp` | POST | Streamable HTTP（2025-11-25，推荐） |
| `/mcp` | GET  | SSE 流（服务端推送） |
| `/message` | POST | 旧 JSON-RPC 端点（保留兼容） |
| `/sse` | GET/POST | 旧 SSE 端点（保留兼容） |
| `/` | GET  | 服务器信息页 |

## 客户端配置示例

### Claude Desktop / Cursor（新 Streamable HTTP）

```json
{
  "mcpServers": {
    "redc": {
      "url": "http://localhost:8089/mcp"
    }
  }
}
```

### 旧版客户端（SSE）

```json
{
  "mcpServers": {
    "redc": {
      "url": "http://localhost:8089/sse",
      "transport": "sse"
    }
  }
}
```

## 未来维护注意事项

1. **新增 2025-11-25 特性**：如需支持 `elicitation`、`tasks`、`structured tool output` 等新特性，需在 `ServerCapabilities` 中声明对应能力，并在 `HandleRequest` 中添加对应 method 处理。

2. **Session Management**：当前未实现 `MCP-Session-Id` 会话管理（无状态服务器）。如需多会话隔离，需在 HTTP handler 中解析/生成 session ID 并隔离 server 实例。

3. **版本列表维护**：MCP 发布新版本时，在 `supportedVersions` 数组头部追加新版本，并更新 `MCPVersion` 常量即可。

4. **工具列表**：所有工具定义在 `getTools()` 函数中，工具实现在 `executeTool()` 中。添加新工具只需在两处各加一个 case。

5. **MCP 官方规范**：https://modelcontextprotocol.io/specification
