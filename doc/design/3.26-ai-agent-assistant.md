# 3.26 AI Agent 助手模式

## 概述

AI Agent 助手模式是 AI 对话页面中的第五种对话模式，允许用户通过自然语言操作 RedC 的全部功能——场景管理、模板操作、远程命令执行等。底层基于 **OpenAI Function Calling** 协议和 RedC 已有的 **MCP 工具体系**，无需额外启动 MCP 服务。

## 架构设计

```
┌──────────────┐     自然语言      ┌──────────────────┐
│   用户输入    │ ──────────────→  │  AgentChatStream  │
│  (前端)      │                  │  (app_ai_chat.go) │
└──────────────┘                  └────────┬─────────┘
                                           │
                                  ┌────────▼─────────┐
                                  │   Agentic Loop    │
                                  │  (最多10轮)       │
                                  └────────┬─────────┘
                                           │
                              ┌────────────┼────────────┐
                              ▼            ▼            ▼
                     ChatWithTools    ExecuteTool   最终回复
                     (OpenAI API)    (MCP Server)  (流式输出)
```

### 核心流程

1. 用户在 Agent 模式下发送消息
2. 前端调用 `AgentChatStream(conversationId, messages)`
3. 后端构建系统提示词 + 历史消息 + MCP 工具定义
4. 进入 Agentic 循环（最多 10 轮）：
   - 调用 `ChatWithTools()` 发送至 OpenAI（非流式，含 `tools` 参数）
   - 若返回 `tool_calls`：
     - 发射 `ai-agent-tool-call` 事件（前端显示⏳工具卡片）
     - 通过 `MCPServer.ExecuteTool()` 本地执行工具
     - 发射 `ai-agent-tool-result` 事件（前端更新为✅/❌）
     - 将工具结果追加到消息历史，继续下一轮
   - 若无 `tool_calls`：输出最终文本回复，逐字符流式发射 `ai-chat-chunk`
5. 发射 `ai-chat-complete`，前端保存消息到历史

## 关键技术细节

### 1. MCP 工具直接调用（不依赖 MCP 服务器）

Agent 模式 **不需要** 用户启动 MCP 服务器。后端直接实例化 `mcp.NewMCPServer(project)` 对象，调用其 `GetTools()` 和 `ExecuteTool()` 方法。这些方法是对内部 `getTools()` 和 `executeTool()` 的公共封装：

```go
// mod/mcp/mcp.go
func (s *MCPServer) ExecuteTool(name string, args map[string]interface{}) (ToolResult, error) {
    return s.executeTool(name, args)
}

func (s *MCPServer) GetTools() []Tool {
    return s.getTools()
}
```

### 2. OpenAI Function Calling 格式

请求中通过 `tools` 数组传递工具定义，`tool_choice: "auto"` 让模型自主决定是否调用工具：

```go
type ToolDefinition struct {
    Type     string         `json:"type"`     // "function"
    Function ToolFunctionDef `json:"function"`
}

type ToolFunctionDef struct {
    Name        string                 `json:"name"`
    Description string                 `json:"description"`
    Parameters  map[string]interface{} `json:"parameters"`
}
```

响应中若包含 `tool_calls`，每个 tool call 有唯一 `id`，工具结果需以 `role: "tool"` + `tool_call_id` 回传：

```go
aiMessages = append(aiMessages, ai.Message{
    Role:       "tool",
    Content:    resultContent,
    ToolCallID: tc.ID,
    Name:       tc.Function.Name,
})
```

### 3. 系统提示词设计

`AgentSystemPrompt` 包含以下关键指导：

- **能力描述**：场景管理、模板管理、远程操作、配置检查
- **工作原则**：
  - 模糊指令先用 `list_cases` 获取信息
  - 破坏性操作需明确告知用户
  - 链式操作自动完成（如查 ID → 停止）
- **case_id 说明**：强调场景操作用 64 字符哈希 ID，非名称；用户给名称时自动查找

### 4. 前端组件持久化

AIChat 组件被渲染在 App.svelte 的 `{#key activeTab}` 块 **外部**，使用 `display:none/block` 控制可见性。这确保：

- 切换标签页时组件不被销毁，事件监听器始终存活
- Agent 长时间执行时，用户切走再切回，结果不丢失
- `onDestroy` 负责清理（而非 `onMount` 的 return）

```svelte
<!-- App.svelte -->
<div style:display={activeTab === 'aiChat' && !isLoading ? 'block' : 'none'}>
  <AIChat {t} onTabChange={...} visible={activeTab === 'aiChat' && !isLoading} />
</div>
```

`visible` prop 用于触发 `$effect()` 检查待处理的终端文本（SSH 终端 → AI 对话功能）：

```svelte
$effect(() => {
  if (visible) {
    checkPendingTerminalText();
  }
});
```

### 5. 工具卡片 UI

前端为每次工具调用渲染状态卡片：

| 状态 | 图标 | 样式 |
|------|------|------|
| `calling` | ⏳ 旋转动画 | `bg-amber-50 border-amber-200` |
| `success` | ✅ | `bg-emerald-50 border-emerald-200` |
| `error` | ❌ | `bg-red-50 border-red-200` |

工具名称通过 `toolNameMap` 映射为友好中文名称（如 `list_cases` → `🔍 列出场景`）。

工具卡片在两处渲染：
- **实时**：`agentToolCalls` 状态变量，在 streaming indicator 区域
- **历史**：消息对象的 `toolCalls` 属性，在 assistant 消息气泡上方

### 6. 事件流

```
Backend                              Frontend
   │                                    │
   ├── ai-agent-tool-call ──────────→  显示⏳工具卡片
   │   {toolCallId, toolName, toolArgs}
   │                                    │
   ├── ai-agent-tool-result ────────→  更新为✅/❌
   │   {toolCallId, success, content}
   │                                    │
   │   ... (可能多轮) ...               │
   │                                    │
   ├── ai-chat-chunk ───────────────→  逐字追加最终回复
   │   {conversationId, chunk}
   │                                    │
   └── ai-chat-complete ───────────→  保存消息，结束 streaming
       {conversationId, success}
```

### 7. 安全保护

- **最大轮次**：10 轮工具调用上限，防止无限循环
- **超时**：5 分钟 `context.WithTimeout`
- **提示词约束**：破坏性操作前需告知用户
- **错误隔离**：单个工具执行失败不终止整个流程，错误信息追加到消息历史供 AI 参考

## SSH 终端 → AI 对话联动

SSH 终端页面支持选中文本后一键发送到 AI 对话分析：

1. SSH Manager 将选中文本写入 `localStorage('ai-chat-pending-terminal')`
2. 切换到 AI 对话页面时，`$effect()` 检测 `visible` 变化
3. `checkPendingTerminalText()` 读取并清除 localStorage
4. 自动切换到自由对话模式，将文本填入输入框（带分析提示前缀）

## 维护注意事项

1. **添加新 MCP 工具**：只需在 `mod/mcp/mcp.go` 中注册工具，Agent 会自动获取并使用
2. **修改提示词**：编辑 `mod/ai/prompt.go` 的 `AgentSystemPrompt`
3. **修改工具名称映射**：编辑 `AIChat.svelte` 中的 `toolNameMap` 对象
4. **组件生命周期**：AIChat 在 App 生命周期内只创建一次，`onMount` 只执行一次；需要在标签页切换时触发的逻辑应放在 `$effect()` 中监听 `visible` prop
5. **Svelte 5 注意**：使用 `$state()` / `$derived()` / `$effect()` / `$props()`，不要使用 Svelte 4 的 `let` 响应式声明或 `$:` 语句
