# 3.18 Wails 构建与 CI/CD 问题修复

## 1. 问题描述

在 GitHub Actions 中构建 redc-gui 时遇到以下问题：

### 1.1 Wails 绑定文件缺失

```
Could not resolve "../wailsjs/runtime/runtime.js" from "src/App.svelte"
```

前端构建失败，找不到 Wails 运行时文件。

### 1.2 Release 上传失败

```
❌ Too many retries. Aborting...
Error: Too many retries.
```

GitHub Release 上传步骤失败，Windows 和 macOS 的 artifacts 上传冲突。

---

## 2. 问题根因

### 2.1 绑定文件缺失

1. `frontend/wailsjs/` 目录被添加到 `.gitignore` 中
2. GitHub Actions 中没有该目录，导致 Vite 构建失败
3. Wails 构建虽然会自动生成绑定，但前端构建在绑定生成之前就失败了

### 2.2 Release 上传冲突

1. macOS 和 Windows 的 artifacts 下载到同一个 `dist` 目录
2. 两个 artifacts 文件名相同（取决于打包方式），导致覆盖或冲突

---

## 3. 修复方案

### 3.1 修复绑定文件问题

**修改 `.gitignore`**:

```diff
# Wails
build/bin/
frontend/node_modules/
frontend/dist/
- frontend/wailsjs/
+ # frontend/wailsjs/  # 注释掉，允许提交到仓库
```

**确保目录结构正确**:

```
frontend/wailsjs/
├── go/
│   ├── main/
│   │   ├── App.js
│   │   └── App.d.ts
│   └── models.ts
└── runtime/
    ├── runtime.js
    ├── runtime.d.ts
    └── package.json
```

**wails.json 配置**:

```json
{
  "wailsjsdir": "./frontend/wailsjs"
}
```

### 3.2 修复 Release 上传问题

**分目录下载 artifacts**:

```yaml
- name: Download macOS Artifact
  uses: actions/download-artifact@v4
  with:
    name: redc-gui-macos
    path: dist/macos

- name: Download Windows Artifact
  uses: actions/download-artifact@v4
  with:
    name: redc-gui-windows
    path: dist/windows
```

**分别上传**:

```yaml
- name: Release
  uses: softprops/action-gh-release@v2
  with:
    files: |
      dist/macos/*
      dist/windows/*
    generate_release_notes: true
    overwrite: true
```

## 4. GitHub Actions 工作流

完整的工作流配置参考 `.github/workflows/release-gui.yml`:

```yaml
name: Release GUI

on:
  push:
    tags:
      - 'v*'
  workflow_dispatch:

jobs:
  build-macos:
    runs-on: macos-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Build
        run: wails build -platform darwin/universal
        env:
          CGO_ENABLED: 1

  build-windows:
    runs-on: windows-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Build
        run: wails build -platform windows/amd64
        env:
          CGO_ENABLED: 1

  release:
    needs: [build-macos, build-windows]
    runs-on: ubuntu-latest
    steps:
      - name: Download macOS
        uses: actions/download-artifact@v4
        with:
          name: redc-gui-macos
          path: dist/macos

      - name: Download Windows
        uses: actions/download-artifact@v4
        with:
          name: redc-gui-windows
          path: dist/windows

      - name: Release
        uses: softprops/action-gh-release@v2
        with:
          files: |
            dist/macos/*
            dist/windows/*
          overwrite: true
```

---

## 5. 注意事项

### 5.1 本地开发

```bash
# 本地构建
wails build

# 或开发模式
wails dev
```

Wails 会自动生成 `frontend/wailsjs/` 目录。

### 5.2 提交规范

- **不要**忽略 `frontend/wailsjs/` 目录
- 如果目录结构有问题，删除后重新运行 `wails build`

### 5.3 Windows 构建

Windows 构建需要在 PowerShell 环境下运行，注意命令兼容性。
