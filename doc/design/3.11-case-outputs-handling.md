# 场景输出特殊处理

## 概述

本文档记录 RedC 中对场景输出 (Outputs) 的特殊处理逻辑，供后续维护参考。

---

## 1. 输出获取入口

### 1.1 函数位置

文件: `app.go`

```go
func (a *App) GetCaseOutputs(caseID string) (map[string]string, error)
```

### 1.2 处理流程

1. **权限检查**: 获取项目锁，检查项目是否加载
2. **场景查找**: 根据 caseID 从项目获取场景实例
3. **状态检查**: 只处理状态为 `running` 的场景
4. **获取 Terraform 输出**: 调用 `c.TfOutput()` 获取原始输出
5. **值处理**: 移除 JSON 字符串值的引号
6. **路径解析**: 将相对路径转换为绝对路径
7. **模块特殊处理**: 根据场景模块添加额外的输出信息

---

## 2. 相对路径解析

### 2.1 处理逻辑

对于 Terraform 输出中的相对路径，会自动解析为绝对路径：

- `.\/` → 基于场景目录的相对路径
- `..\/` → 基于场景目录的父级相对路径

### 2.2 代码实现

```go
if isRelativeFilePath(value) {
    absPath := filepath.Join(c.Path, value)
    if _, err := os.Stat(absPath); err == nil {
        value = absPath
    }
}
```

## 3. 模块特殊输出处理

### 3.1 gen_clash_config / upload_r2 模块

当场景使用了 `gen_clash_config` 或 `upload_r2` 模块时，会自动添加以下输出：

#### 触发条件

```go
if c.Module != "" && (strings.Contains(c.Module, "gen_clash_config") || strings.Contains(c.Module, "upload_r2")) {
```

#### 添加的输出

| 输出键 | 说明 | 来源 |
|--------|------|------|
| `clash_config_local` | 本地配置文件路径 | 场景目录下的 `config.yaml` |
| `clash_config_r2` | R2 存储路径 | 格式: `r2:{bucket_name}/{filename}` |

#### 配置读取

从 `terraform.tfvars` 文件读取以下配置：

```hcl
filename    = "aliyun-config.yaml"    # 文件名
buckets_name = "test"                 # R2 存储桶名称
buckets_path = "path/to"              # R2 存储路径（可选）
```

#### R2 路径格式

- 无 buckets_path: `r2:{bucket_name}/{filename}`
- 有 buckets_path: `r2:{bucket_name}/{buckets_path}/{filename}`

### 3.2 模板示例

`redc-template/aliyun/proxy/case.json`:

```json
{
  "name": "proxy",
  "redc_module": "gen_clash_config,upload_r2",
  ...
}
```

## 4. 如何添加新的模块特殊输出

### 4.1 步骤

1. 在 `GetCaseOutputs` 函数中添加条件判断
2. 读取所需的配置文件（如 terraform.tfvars）
3. 将额外输出添加到 `result` map 中

### 4.2 示例代码

```go
// 检查模块是否包含特定模块
if c.Module != "" && strings.Contains(c.Module, "your_module_name") {
    // 读取配置文件
    configPath := filepath.Join(c.Path, "config.yaml")
    if _, err := os.Stat(configPath); err == nil {
        // 读取配置并处理
        // ...
        
        // 添加输出
        result["your_output_key"] = yourValue
    }
}
```

### 4.3 注意事项

- 模块名称检查使用 `strings.Contains`，支持逗号分隔的多个模块
- 添加输出前检查文件是否存在，避免错误
- 使用 `c.Path` 获取场景目录路径
- 处理好空值和默认值

## 6. 注意事项

1. **模块名称**: 模板中 `case.json` 使用 `redc_module` 字段，代码中检查时注意名称匹配
2. **文件路径**: 使用 `filepath.Join` 确保跨平台路径兼容
3. **错误处理**: 文件不存在时静默跳过，不影响其他输出
4. **输出格式**: Terraform 输出值可能带引号，需要移除
